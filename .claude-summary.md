# 项目概述

> CCTV-17 农业农村频道收视日报 PPT 自动生成器

## 项目名称

CCTV-17 收视日报 PPT 生成器 (ppt-generator)

## 项目描述

自动读取 Excel 收视数据，生成精美的 CCTV-17 频道每日收视报告 PPT。支持 6-sheet 基础格式和 12-sheet 完整格式两种 Excel 输入，分别生成 11 页和 17 页的报告。

## 技术栈

- Python 3.11
- python-pptx 1.0.2 (PPT 生成与操作)
- openpyxl (Excel 读取)
- lxml (XML 级别 PPT 内容替换)
- PPT 尺寸: 13.33×7.5 英寸宽屏 (Emu 12192000×6858000)
- 模板: origin.pptx (3 种布局: 普通/1_普通/空白)
- Demo 模板: `output_ppt/CCTV-17频道收视日报 0210 demo（简版）.pptx`

## 目录结构

```
ppt_config.py                      # 主题配置(颜色/字体/布局常量)
data_reader.py                     # Excel 数据读取与解析(13个dataclass，11个读取函数)
slide_utils.py                     # 图表/表格/文本工具函数
generate_report.py                 # 基础生成器(17种幻灯片构建函数+CLI入口)
generate_report_config_driven.py   # v3 config-driven生成器(XML级别，格式完美复刻demo)
origin.pptx                        # PPT 模板文件
input_data/                        # Excel 输入数据目录
output_ppt/                        # PPT 输出目录
```

## 使用方法

```bash
# 推荐: v3 config-driven 模式（XML级别复刻demo格式）
python -X utf8 generate_report_config_driven.py "input_data/20260215CCTV-17日数据（简版）.xlsx"

# 基础模式(从零构建PPT)
python -X utf8 generate_report.py "20260210CCTV-17日数据.xlsx" --template origin.pptx
```

## 架构说明

### v3 Config-Driven 生成器 (`generate_report_config_driven.py`)

核心思路: **复制 demo → lxml XML 级别替换 `<a:t>` 文本 → 保留所有原始格式**

1. 调用 `generate_report.py` 生成"数据草稿"（含正确数据，格式粗糙）
2. 复制 demo 文件作为输出基础
3. 逐页 XML 回填: 用草稿数据覆盖 demo 的旧文本
4. 补全标题: 对 pages 3-8 用 Excel 数据构建标题文本
5. 保存最终文件

关键技术:
- **命名空间**: `p:txBody`(形状) vs `a:txBody`(表格单元格)
- **两轮匹配**: 先内容精确匹配(短文本≤30字符，防标签错位)，再位置最近邻匹配
- **非空段落对齐**: demo 有空行分隔符，草稿无空行，只对非空段落做 1:1 映射
- **`\x0b`/`\n` 规范化**: demo 用 `<a:br>`(→`\x0b`)，草稿用段落分隔(`\n`)
- **Page 13 专用**: 动态定位变化百分比文本框至柱形图上方
  - `_sync_and_position_audience()`: 同步图表数据+计算几何位置+更新标签
  - 19类别结构(性别2+年龄7+教育5+城乡2 + 3个Gap间隔)
  - `_compute_nice_axis()`: PowerPoint风格坐标轴上限自动计算
  - XML后处理: 清除`replace_data`产生的"None"间隔类别
  - 标签分类: ↑↓前缀=legend, 数字/-前缀=bar label
- **市场份额3类别**: 图表使用"前1个月均值/前一日/当日"3个类别，匹配demo结构
- **变化列安全读取**: Excel列6+为观众规模数据，非变化百分比；代码检查header含"变化"才覆盖计算值
- **描述文本双段落**: `build_market_share()`生成2个独立`<a:p>`段落，确保v3 sync不留demo残留
- **分分钟3表结构**: 每个分分钟slide含时段标签表+节目标签表+空的前月剧集标签表(1r×8c)
- **分分钟首播变色**: 节目标签表着色依据串单P列(首重播)实际数据，P列="首播"才变色（收视率页橙色F39C12/份额页绿色4A7C31）；ProgramItem.premiere_type字段存储该值
- **速报文本用词**: 排名变化描述统一用「较前一日提升/下滑x位」或「较前一日保持不变」；两个文件的`_rank_change_desc`已同步（"上升"→"提升"）
- **分分钟时段标签对齐**: `_fix_minute_chart_page()`中表头/分隔线表格保持模板原始left/width，列宽按图表时段边界位置计算（col[0]=表左~10:00, col[i]=boundary[i]~boundary[i+1], col[7]=22:30~表右），精确对齐横坐标刻度
- **分分钟z-order修复**: 确保节目标签表格(20/21c)始终在分隔线表格(8c)之上，解决Page10节目名称不可点击编辑的问题（demo模板原始bug）

### 基础生成器 (`generate_report.py`)

- 从零构建 PPT（加载 origin.pptx → 删除所有幻灯片 → 程序化重建）
- 17 种幻灯片构建函数
- `BODY_PLAIN_MODE`: 简版模板时跳过标题栏生成

## 开发说明

- 使用 `-X utf8` 避免 Windows GBK 编码问题
- 字体: 微软雅黑 (Microsoft YaHei)
- 图表类型: COLUMN_CLUSTERED, BAR_CLUSTERED, AREA
- 忠实度计算: loyalty = rating / (audience_万人/universe_万人*100) * 100
