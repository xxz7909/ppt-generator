## 项目简介

本工具解决了每日手工制作收视报告 PPT 耗时的问题：

- **输入**：当日收视数据 Excel 文件（含 6 个或 12 个 sheet）
- **输出**：与 demo 模板格式完全一致的 PPT（11 页简版 / 17 页完整版）
- **核心技术**：以 demo 模板 pptx 为基础，通过 `lxml` 在 XML 层面精确替换数据，100% 保留模板的字体、颜色、布局等所有原始格式

## 文件说明

### 核心运行文件

| 文件 | 说明 |
|---|---|
| `GUI.py` | **图形界面入口**。基于 tkinter 实现，支持拖拽选择 Excel 文件、自动检测模板、推导输出文件名、记忆上次配置、实时显示生成日志 |
| `generate_report_config_driven.py` | **v3 配置驱动生成器（主逻辑）**。复制 demo 模板 → lxml XML 级逐页回填数据 → 保存。包含所有页面的特殊处理逻辑（颜色修正、动态定位、坐标轴自适应等） |
| `generate_report.py` | **数据草稿生成器（辅助）**。从零构建含正确数值的"数据稿" PPT，供 v3 生成器读取数据用。不直接产出最终文件 |

| `ppt_config.py` | **主题配置常量**。定义颜色方案（`Colors`）、字体参数（`Fonts`）、布局参数（`Layout`）、幻灯片尺寸等全局常量 |
| `slide_utils.py` | **幻灯片构建工具库**。封装文本框、富文本框、标题、图表（柱/条/面积）、表格等构建函数，供 `generate_report.py` 调用 |

## 项目架构与生成逻辑

### 总体架构

```
Excel 数据文件
      │
      ▼
[generate_report.py]  ──→  "数据草稿" PPT（临时文件）
  从零构建，数值正确，格式粗糙

      │
      ▼  （v3 配置驱动器读取草稿数值）

  demo 模板 .pptx ──→ 复制 ──→ 输出文件（初始与模板完全相同）
                                     │
                              逐页 XML 回填
                                     │
                         ┌───────────┼───────────┐
                      文本框      图表数据      表格单元格
                   仅替换<a:t>  replace_data  逐格替换文本
                   保留所有rPr   保留numFmt    保留表格样式
                                     │
                              页面专项修正
                              (颜色/坐标轴/定位)
                                     │
                                     ▼
                            最终输出 PPT ✅
```

### 关键技术细节

#### 1. XML 级文本替换

系统使用 `lxml` 直接操作 `<a:t>` 标签（纯文字节点），不触碰 `<a:rPr>`（字体/颜色/字号），从而保留模板的所有样式：

```
<a:r>
  <a:rPr lang="zh-CN" sz="1400" b="1">   ←── 保留不动
    <a:solidFill><a:srgbClr val="FF0000"/></a:solidFill>
  </a:rPr>
  <a:t>0.123</a:t>   ←── 只替换这里
</a:r>
```

#### 2. 两种形状匹配算法

- **静态标签**（文字≤30字符）：先做**内容精确匹配**，防止静态标签被数据错误覆盖
- **数据形状**：按**最近邻位置匹配**（欧氏中心距离最小）将草稿形状与模板形状配对
- **图表**：按 **Y 坐标排序**后一一配对（解决条形图交叉错配问题）

#### 3. 非空段落对齐

demo 文本框内常有空行分隔段落，草稿没有。系统提取两端的"非空段落"后按顺序 1:1 映射，保留 demo 原有的空行结构。

#### 4. 数据标签数字格式恢复

`replace_data` 会重置图表的 `numFmt`，系统在替换前先保存模板中的格式码，替换后恢复。

### 自动处理逻辑

- ✅ 封面日期从 Excel 文件名自动提取（`YYYYMMDD`）
- ✅ 副标题"第 N 期"自动计算（当年第几天）
- ✅ 变化描述口径统一（≤5% 基本持平 / >5% 提升或下降）
- ✅ 排名变化描述（上升 N 位 / 下滑 N 位 / 保持不变）
- ✅ 第3页变化列颜色（提升红/下降绿）
- ✅ 第4页 CCTV-17 行红色高亮 + 阈值线动态定位
- ✅ 第5页 CCTV-17 行高亮 + 箭头颜色标注
- ✅ 第7/8页超轴最大值标注
- ✅ 第11/12页 Y 轴自适应扩缩
- ✅ 第13页变化标签动态定位至柱顶
- ✅ 自动清除模板中遗留的 PowerPoint 批注

## 源代码使用方法

### 1. 创建虚拟环境并安装依赖

```powershell
cd D:\Workspace\ppt-generator

# 创建虚拟环境
python -m venv .venv

# 激活虚拟环境
.\.venv\Scripts\Activate.ps1

# 安装依赖
pip install python-pptx openpyxl lxml pandas pillow
```

> `requirements.txt` 中列出的包含开发/调试用工具（xlwings、selenium 等），生成 PPT 只需上述 5 个包。

### 2. 准备文件

```
ppt-generator/
├── input_data/
│   └── 20260215CCTV-17日数据（简版）.xlsx   ← 放入 Excel 文件
└── output_ppt/
    └── CCTV-17频道收视日报 0210 demo（简版）.pptx  ← 放入 demo 模板
```

### 3. 方式一：GUI 界面（推荐）

```powershell
python -X utf8 GUI.py
```

界面操作：
1. 点击"浏览"或将 Excel 文件拖入输入框
2. 模板路径会自动检测（`output_ppt/` 下含 "demo" 的 pptx），也可手动选择
3. 输出文件名自动推导为 `CCTV-17频道收视日报 MMDD.pptx`，可手动修改
4. 点击"生成 PPT"，日志区实时显示进度
5. 生成完成后点击"打开输出目录"查看结果

### 4. 方式二：命令行（v3 配置驱动，推荐）

```powershell
python -X utf8 generate_report_config_driven.py `
    "input_data/20260215CCTV-17日数据（简版）.xlsx" `
    --template "output_ppt/CCTV-17频道收视日报 0210 demo（简版）.pptx" `
    --output "output_ppt/CCTV-17频道收视日报 0215.pptx"
```

运行后控制台显示：
```
[1/4] 生成数据稿: input_data/20260215...
[2/4] 复制模板: output_ppt/... demo... → output_ppt/...0215.pptx
[3/5] XML 级精确回填...
  - 第 1/11 页
  - 第 2/11 页
  ...
[4/5] 修复标题...
[5/5] 保存: output_ppt/CCTV-17频道收视日报 0215.pptx
✅ 完成！共 11 张幻灯片
```

> `-X utf8` 参数在 Windows 下避免 GBK 编码错误，**必须加上**。

### 5. 方式三：命令行（基础模式，仅生成数据稿）

```powershell
python -X utf8 generate_report.py `
    "input_data/20260215CCTV-17日数据（简版）.xlsx" `
    --template origin.pptx `
    --output "output_ppt/draft_0215.pptx"
```

> 此模式只用于调试数据，最终样式不如 v3 模式精确，不建议日常使用。

### 6. 更新模板阈值配置

当 demo 模板中阈值线数值或颜色发生变化后，重新提取配置：

```powershell
python -X utf8 extract_demo_config.py `
    "output_ppt/CCTV-17频道收视日报 0210 demo（简版）.pptx" `
    --output demo_layout_config.json
```

## 构建 exe 可执行文件

### 前提条件

```powershell
# 安装 PyInstaller
pip install pyinstaller
```

### 执行构建

```powershell
# 在项目根目录下运行
pyinstaller ppt_generator.spec
```

构建完成后，在 `dist/` 目录中会生成 `CCTV17收视日报生成器.exe`（单文件 exe，约 40-80 MB）。

### 打包产物部署

将以下文件复制到目标机器的同一目录下即可：

```
部署目录/
├── CCTV17收视日报生成器.exe   ← 主程序
├── demo_layout_config.json    ← 阈值配置（从项目根目录复制）
├── input_data/                ← 存放 Excel 输入文件
└── output_ppt/
    └── CCTV-17频道收视日报 0210 demo（简版）.pptx  ← demo 模板
```

> **注意**：demo 模板 `.pptx` 不内嵌于 exe 中，需与 exe 一同部署在 `output_ppt/` 目录（或 exe 同级目录下，GUI 会自动检测）。

## 模板文件定制

详见 [模板文件定制指南.md](模板文件定制指南.md)。

简要原则：
- **改模板直接生效**：字体、颜色、字号、位置、图表样式、背景、装饰图形、最后一页所有内容
- **代码强制覆盖**：各页数据数值、CCTV-17 行高亮颜色、第13页标签位置
- **可间接定制**：第4页阈值线颜色和数值（修改模板中 `文本框 10` 后重新提取配置）

## 输入 Excel 格式说明

### 简版（6 个 sheet）→ 生成 11 页 PPT

| Sheet 名 | 内容 |
|---|---|
| `市场份额` | 频道收视率、市场份额、到达率、忠实度（CCTV-17 及全频道均值），含前月均值和前一日数据。H:W 列为观众规模分类数据（第13页图表数据源） |
| `上星频道排名` | 各上星频道当日/前一日市场份额排名 |
| `台组内排名` | 央视台组内各频道排名及份额 |
| `电视剧与非电视剧` | 电视剧/非电视剧的当日与前月均值份额 |
| `串单` | 当日完整节目单（开始/结束时间、节目名、分类、收视数据） |
| `分分钟` | 从 05:30 到 25:59 每分钟的收视率和市场份额 |

### 完整版（12 个 sheet）→ 生成 17 页 PPT

在简版基础上额外包含历史对比等更多 sheet（详见 `data_reader.py` 中的读取函数）。

### 文件名命名要求

文件名中**必须包含 8 位日期**（格式 `YYYYMMDD`），程序从文件名提取日期：

```
✅ 20260215CCTV-17日数据（简版）.xlsx
✅ CCTV-17_20260215_日数据.xlsx
❌ CCTV-17日数据.xlsx    （无日期，生成的封面日期将不正确）
```
