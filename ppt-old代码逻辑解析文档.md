# `ppt-old.py` 逻辑解析文档

## 1. 脚本定位

`ppt-old.py` 是旧版的 CCTV-17 日报 PPT 生成脚本，核心流程是：

1. 读取 `origin.pptx` 模板
2. 读取 `{date}.xlsx`（例如 `0219.xlsx`）
3. 从多个 sheet 提取“前一月均值 vs 当日值”
4. 按固定页码和固定 shape 索引回填文字、表格、图表
5. 输出 `{date}频道收视日报.pptx`

入口：

```python
if __name__ == '__main__':
    main(sys.argv[1:])
```

调用方式：

```bash
python -X utf8 ppt-old.py 0219
```

---

## 2. 输入/输出与依赖

### 2.1 输入文件

- PPT 模板：`origin.pptx`
- Excel 数据：`{date}.xlsx`（例如 `0219.xlsx`）

### 2.2 输出文件

- `./{date}频道收视日报.pptx`

### 2.3 关键 Python 依赖（旧脚本兼容性）

- `python-pptx`
- `xlrd`（脚本使用 `xlrd.open_workbook` 打开 `.xlsx`）
- `pandas`
- `numpy`

> 兼容性注意：
>
> - `xlrd>=2.0` 不支持 `.xlsx`，旧脚本要求 `xlrd==1.2.0` 这类旧版本。
> - 脚本使用 `DataFrame.append`，在 `pandas>=2.0` 已移除，需 `pandas<2.0` 或改代码。

## 3. 全局变量与工具函数

### 3.1 全局变量

- `date = '0000'`：在 `main` 中被 `argv[0]` 覆盖，所有文本标题依赖它。

### 3.2 文本/变化计算函数

1. `str_date(date)`
   - 把 `0219` 转成 `02月19日`

2. `str_p_date(date)`
   - 计算“前一日”文本，处理跨月边界（如 3 月 1 日 -> 02 月 28 日）

3. `increase_rank(p, n)`
   - 排名变化描述：`提升x位 / 下滑x位 / 保持不变`

4. `increase_percent(p, n)`
   - 百分比变化描述：`提升x% / 下滑x% / 保持不变`

### 3.3 PPT 填充函数

1. `chart_title_text(shapes, shape, p, n)`
   - 给图表标题写入 `▲/▼` 与变化比例
   - 并替换图表数据为 `['前1月均值', 当日]` 两类

2. `table_data(shapes, shape, p, n)`
   - 给 2 列表格填充变化箭头、前1月均值和当日值

3. `rating_increase(shapes, increase, col)`
   - 在第10页的分时段统计表中写入 `↑/↓/持平`

## 5. 按页生成内容（共 10 页）

> 说明：脚本大量使用 `slides[i].shapes[j]` 的固定索引写内容，强依赖模板版式，模板一改就可能错位。

### 第4页（slide index 3）- 台组内排名对比表

- 标题写“今日排名 + 较前一日变化”

### 第5页（slide index 4）- 上星频道排名对比表

### 第6页（slide index 5）- 串单双指标图

- X 轴分类：`栏目名 + 开始时间`
- 双序列：市场份额%、收视率%
- 若收视率峰值较高，动态抬高 Y 轴上限
- 对 `国歌/再见` 做特殊重排（用于后续页面）

### 第7页（slide index 6）- 串单市场份额排名图

### 第9页（slide index 8）- 分分钟市场份额

## 6. 旧脚本的强耦合点与风险

1. **模板耦合**：依赖固定 shape 索引（如 `shapes[17]`），模板稍改即错位。
2. **表头耦合**：依赖特定中文列名（如 `市场份额%.1`、`单位 >>`、`频道调整`）。
3. **时间格式耦合**：`分分钟` 页直接按字符串匹配 `06:00:00`、`1900-01-01 00:00:00`。
4. **旧依赖耦合**：`xlrd` 与 `pandas.append` 都是旧生态写法。

## 7. 使用 `0219.xlsx` 的实测结论

已执行：

```bash
python -X utf8 ppt-old.py 0219
```

实测过程中的关键结论：

1. `0219.xlsx` 能作为输入被读取；前置 sheet 处理可运行。
2. 需满足旧依赖兼容：
   - `xlrd==1.2.0`（否则 xlsx 不支持）
   - `pandas<2.0`（否则 `DataFrame.append` 报错）
3. 在“第10页 分分钟收视率”阶段会报错：
   - 代码查找 `df_copy['单位 >>'] == '1900-01-01 00:00:00'`
   - 当前数据中该字段实际表现为类似 `1 day, 1:59:00`（时间跨天格式）
   - 导致 `IndexError: index 0 is out of bounds`

结论：

- `0219.xlsx` **可作为旧脚本输入进行测试**，但在当前环境下默认**无法完整跑通到最终输出**，主要卡在“24:00 时间字符串匹配”这一旧逻辑假设。

## 8. 最小修复建议（不改业务逻辑）

若只为让旧脚本稳定跑通，可做最小改造：

1. 把 `xlrd` 读取改为 `pandas/openpyxl` 路径，避免 `xlrd` 版本问题。
2. 把 `df.append(...)` 改为 `pd.concat(...)`。
3. 对 `分分钟` 的时间字段先标准化（转 `datetime.time` 或分钟索引），不要硬编码匹配 `1900-01-01 00:00:00`。

这样能保留旧版业务口径，同时降低环境差异导致的失败率。
